1. On-Demand Instances : Default pricing model in AWS.

Pay only what you use.
No long-term commitment
Changed per Second basis (Min 60 Sec)
Highest flexibility and Highest Cost

Best for : Testing, short duration runs, Temp environment (dev/qa/uat)


2. Reserved Instances (RI)

Commitment required for 1 Yr / 3 yrs
We get upto ~70% discount compared to On-Demand.

	2.1 Standard RI : Maximum Discount, Less Flexibility to change configuration (Fixed Instance family/type)
	2.2 Convertable RI : Lower Discount, More flexibility to change Instance Family / OS / Tenancy.

Payment Options for RI : 
	All Upfront : Pay everything at the time of purchase, No monthly cost.
	Partial Upfront : YOu pay 30-60% now, AWS Decrease the hourly cost of the instance.. Monthly cost occurs, 	but with redused cost..
	No Upfront : Pay everything monthly basic.. Cost similar to On-Demand..


3. Saving Plans : THis is modern replacement/improvement over RIs. AWS intrododused this to reduse the RIs complexity. you pay it whether you use it or not

We are commiting with AWS on Hourly price. You are giving commitment to aws to spend 10$/Hr, so AWS offers good amount of discount.

	3.1 Compute Saving Plan : For multiple services : ec2, lambda, fargate.. We can change Instance fanily / size / Region / OS / AZs.. 
	
	3.2 EC2 instance savings plan : Less flexible but good discount.. Applicable to only EC2 service.. We can change Instance family in a specific region..

3.3 SageMaker Saving Plan : AI/ML related workload 
3.4 Database Saving Plan : Database realted worklaod


4. Spot Instances : AWS Offers its unused capacity at huge discount. BUT ..... AWS Can terminate/Delete the instance anytime with Just 2 min warning. 
Batch Job, Rendering, Temp Jobs, Easily recoverable jobs, Interruptable jobs..

--> If Price Not increase, YOU Terminated our INstance, We need to pay full amount..
--> If price increased, AWS terminated our instance, we get charged for FUll hours only, We dont get charged for Partial hours..

5 hrs 50 Minutes.. 

YOU Terminate : 5 hrs 50 min
AWS Terminate : 5 hrs only

Not good for Production workload / DBs / Stateful systems.. 

--> We have to bid against aws pricing. if our bid price is more than or equal, AWS provides the server.

DLM : Datalifecycle manager : 


RTO : Recovery Time Objective : 
Meaning : Downtime limit
The Question : How long can we afford to be offline
Focus: Recovery speed
Example: if our RTO is 2 hours, We must have systems should be backup - "up and running" with in 2 hours to avaoid critical business damage.

RPO : Recovery Point Objective : 
Meaning: Your data loss limit
The Question: How much data loss we can afford.
Focus : Backup frequency
Example: If your RPO is set to 1 hour, you must have backup configured for every hour.

Data Lifecycle Manager : Helps us to automate the creation, retention, copy and deletion of snapshots.

EFS works on 2049 port number. Protocol : NFS (Network File System v 4)

Opening 2049 port at SG Level.. But to where i need to open.

1. Open for EveryOne 				: 0.0.0.0/0		(We dont use this, least secured) XX
2. Open NFS port for Entire vpc 	: 172.31.0.0/16	(good, but problem is Other resources from same VPC can access EFS data)
3. Open NFS port for Instance IPs	: 172.31.100.5/32 & 172.31.100.6/32 (Good, Secured.. But What if we launch a new instance later, We need to edit this SG and need to add entry)

4. Open NFS port on EFS SG to EC2 Instance SG : More secured, I have to make sure newly launched instance are using same SG as existing instances. **** (Explain same in Interviews)
