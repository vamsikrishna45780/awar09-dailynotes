AWS Route53 has 2 types of resolvers, if our organisation is using hybrid-environment.. 
Hybrid : some servers in AWS and Some servers in on-premises.. (site-to-site vpn / aws direct connect)

Inbound DNS resolver : On-Prem to AWS
--> Allows on-premise servers to resolve private records inside the aws vpc..

Flow : on-prem server trying to access aws DB with DNS name.. On-Premise DNS server received the query --> it forwards the query to route53 "Inbound endpoint" --> Send to Route53 HostedZOne --> Verify the record --> Reach the DB Server --> Provides the response to on-premise server..

-----

Outbound DNS resolver : AWS to on-premise

Flow : AWS ec2 instance typoing to access on-prem hosted db.. Request goes to hostedzone "outbound resolver rule" --> Request send to on-premise DNS server --> Then On-Prem DNS server looks for the record mapping --> Sends the req to on-prem db server..
